What is DOM and What is the purpose of DOM?

DOM stands for Document Object Model. It is an interface for programming through which structure, style and content can be access and manipulate of a web document. DOM can interact the web document over HTML and CSS. It represents the whole web document as nodes and object. Generally on condition change we can change the structure, style and content of HTML elements or web pages. DOM is not a programming language, but without it, javascript wouldn't have control on webpages, HTML document, svg document and etc. They all can be accessed and manipulated by DOM

How will you select HTML elements using DOM? Name the DOM methods.

The most common way to access a HTML element is to use 'id' of an element.
Generally HTML elements and all other attributes makes a document within this document we can access and manipulate the HTML structure, style and content by DOM.

There are five ways of selecting element using DOM are followings:

  1. 1. getElementByTagName()
  2. 2. getElementByClassName()
  3. 3. getElementById()
  4. 4. querySelector()
  5. 5. querySelectorAll()
What is event bubble?

Event bubbling is such a concepts that, when an event receives a command to the innermost target element and that events bubble up (or we can say is transmitted or propagated) to its parants untill the root element come through a DOM tree.

The major use of event bubbling is the registration of default functions present in the program

What is a callback function and why will you use it?

When we need a pre-determined function executions after an another function's executions, then we can use a callback function.

Callback function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action. However, callback function often use to continue code executions after an asynchonous operation has completed.
More clearly, A callback function is use or execute on condition completions.
